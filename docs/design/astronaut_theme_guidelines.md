# No-Look-Budget: Astronaut Theme UI/UX Guidelines

本ドキュメントは、「No-Look-Budget」アプリにおける新デザインテーマ「Astronaut（宇宙飛行士）」のUI/UXの一貫性および実装時のルールを定めたガイドラインです。過去の仕様（独自テンキー、立替分離など）や既存の機能（言語設定、グラフの表示数など）はすべてそのまま維持しつつ、参考画像のビジュアルデザインをベースに、宇宙と無重力をテーマにしたスタイルへとアップデートするための指針です。

---

## 1. コンセプト

**「Antigravity（無重力）× No-Look Experience」**

*   **世界観**: ユーザーは限られたリソース（予算）で宇宙を旅する宇宙飛行士。
*   **デザイン言語**: 深宇宙（Deep Space）の暗闇と星座のラインを背景に、HUD（ヘッドアップディスプレイ）のように浮かび上がるモダンスタイル。過度なグロー（発光）を抑え、目がチカチカしないマットな質感を取り入れる。
*   **ADHD・視認性向け**: アプリを開いた瞬間に、直感的な色（落ち着いた緑＝安全・残高、落ち着いた赤＝警告・消費）と巨大な数字で、現在の家計状況を数字を読まなくても「感覚的」に把握できる状態を目指す。

---

## 2. カラーパレット & タイポグラフィ (Dark Mode First)

アプリ全体は**ダークモードを基本（Dark Mode First）**として設計されます。

### 2.1 背景スタイル (Background)
*   **メイン環境 (宇宙背景)**: 深みのあるネイビーからブラックのグラデーション (`#080B14`, `#1A1A2E` 近辺のDeep Space Navy色）。
*   **背景ディテール**: 背景にはかすかな星屑のパーティクルや、線で結ばれた星座（Constellation）の幾何学的なラインをうっすらと敷き、宇宙空間の奥行きを演出する。

### 2.2 グラフ・数値のソリッドカラー (HUD Colors)
グラフや主要な数値は、視認性を確保しつつも目がチカチカしないよう、過度な発光（グロー効果）を抑えたマット（ソリッド）なカラーを使用します。
*   **予算内（残高・安全）**: 落ち着いたミントグリーン (`#22C55E` 〜 `#4ADE80` 近辺で、目に優しい彩度）。
*   **使用済み（消費・危険）**: 落ち着いたコーラルレッド (`#EF4444` 近辺で、目に優しい彩度）。
*   **文字基本色**: `Color.white` （白または淡いシルバー）。
*   **サブ文字色**: `Color.gray`、または透明度を下げて背景に馴染ませた白。

### 2.3 タイポグラフィとレイアウト
*   **フォントスタイル**: クリーンでモダンなサンセリフ体（San Francisco 等のシステム標準フォント）。
*   **数値表示の強調**: 残高や消費額（「¥150,000」など）のメイン数値は、画面内で最も太く（Bold/Heavy）大きく表示し、金額自体に残高（緑）や消費（赤）の色をつけることで直感性を極限まで高める。
*   **ラベル表現**: ラベルテキストは控えめなサイズ・細字にすることで、メインの数値とのコントラストをつける。（※画像内の「REMAINING」「TODAY'S SPENT」などはあくまで見た目の参考とし、実装済みアプリの言語や項目名は変更しないこと）

## 3. UIパーツ・レイアウト構成

**※注意: 以下の構成はあくまで既存の機能とレイアウトにデザインを適用するためのルールです。既存の円グラフの数や表示項目を消したり勝手に変更することは厳禁です。**

### 3.1 メインダッシュボードの象徴 (Astronaut & Rings)
*   **メインリング（巨大ドーナツチャート）**: 画面中央あるいは上部に大きく配置。ゲージの色は緑（残高）と赤（消費）のツートンで構成。
*   **宇宙飛行士（Astronaut）のモチーフ**: メインリングの中央の空きスペースに、無重力を漂う宇宙飛行士のリアルなイラストや画像を配置し、このテーマのアイコンとする。
*   **金額の並列レイアウト**: メインリングの周囲に「全体の残り予算」と「本日の使用額」など、コントラストの強い数値を堂々と配置する。

### 3.2 Glassmorphism (すりガラス効果)
*   **ボタン・カード背景**: メインアクション用のボタン（例: Quick-Sync等のログ追加ボタン）や、各カテゴリを囲む背景枠には、半透明のすりガラス効果（`Ultra Thin Material`）を使用。ベタ塗りを避け、背後の星空や星座ラインが美しく透けるようにする。
*   **形状（カプセル型）**: ボタンや警告アラートのバナーなどは、角の丸みが強いカプセル型（Pill Shape）を用いてモダンさを強調。

### 3.3 アラート表示の徹底
*   予算超過時などの警告バナーは、コンポーネント全体を薄い赤色の半透明で覆い、文字・アイコンも落ち着いた赤色で統一し、目がチカチカしない範囲で危機感を直感的に伝えるデザインとする。

## 4. モーダル・別画面遷移時の注意点（絶対遵守）

SwiftUIにおいて、`.sheet` や `.fullScreenCover` などで別モーダルを呼び出した際、**大元のアプリのカラースキーム（ダークモード指定）が自動で引き継がれないケース**が多発します。

**【ルール】:**
モーダルとして呼び出される全てのView（NavigationStack等の根本）に対して、**必ず `.preferredColorScheme(.dark)` を明示的に修飾してください。**
背景の宇宙用カラーにテキストが同化するバグを防ぎます。

```swift
// 実装例
.sheet(isPresented: $showModal) {
    QuickInputModalView()
        .preferredColorScheme(.dark) // 必須
}
```

## 5. 数値入力ポリシー（Custom NumberPad）

金額などの「数値」を入力させる場面において、**iOS標準のキーボード（TextField等）は絶対に使用しません。**全角半角の混在や予期せぬクリップボードペーストによるクラッシュを防ぐためです。

**【ルール】:**
数値を入力する箇所は、必ず既存の独自のテンキーUIを使用してください。
*   マットなアクセントカラー（Space Green / Warm Orange）を用いて、目が疲れにくく入力アクションを促す美しいUIとすること。

---

## 6. 入力体験（立替と自己支出の2段入力）

「No-Look-Budget」ユニーク機能である「立替金セパレーター」の独自の入力UIを維持します。
*   **フォーカス切り替えと明瞭化**: タップして切り替える2つの入力枠は、現在アクティブな方を「HUDのフォーカス枠」のように枠線のハイライトや背景の明度変化で視覚的に目立たせ（過度な光彩は避ける）、無重力空間に浮かぶインターフェースとしての没入感を保つ。
