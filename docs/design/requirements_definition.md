## 1. プロジェクト概要

### 1.1 背景・目的

浪費家やADHD傾向のユーザーが継続可能な家計管理を実現するiOSアプリ。
既存の家計簿アプリが持つ「入力の面倒さ」「立替による管理崩壊」を解消し、
**アプリを開かなくても予算状況が把握できる（No-Look Experience）** を提供する。

### 1.2 対象ユーザー（ペルソナ）

| ペルソナ | プロフィール | 課題 |
|---------|------------|------|
| メインユーザー | 浪費癖のある20〜40代iPhoneユーザー | 入力が面倒で家計簿が続かない |
| ADHD傾向ユーザー | 注意力の持続が難しいユーザー | アプリを開く行為自体がハードル |
| 立替多発ユーザー | 飲み会幹事、グループ活動担当者 | 立替金が自分の予算と混在して管理崩壊 |

### 1.3 ビジネスゴール

| 目標 | KPI |
|------|-----|
| App Store公開 | 2026年中にリリース |
| マネタイズ | サブスクリプション/機能課金による収益化 |
| ユーザー獲得 | リリース後3ヶ月でDAU 1,000人 |

### 1.4 スコープ定義

| 区分 | 内容 |
|------|------|
| **スコープ内** | 支出記録、立替管理、予算管理、ウィジェット、月次レビュー |
| **スコープ外** | 銀行口座連携、OCRレシート読取、Apple Pay連携、多通貨対応 |
| **将来検討** | プレミアム機能、年次サマリー、AIによる支出分析 |

### 1.5 競合差別化要件

| 差別化ポイント | 競合との違い |
|--------------|------------|
| No-Look Experience | ウィジェットで予算状況を把握。アプリを開かずに管理可能 |
| 立替セパレーター | 立替金を予算と完全分離。2段入力で自己支出額も同時記録 |
| ADHD対応設計 | 最小操作で入力完了。触覚フィードバックで操作の手応えを提供 |

## 2. 機能要件

### FR-001: 支出記録

| 項目 | 内容 |
|------|------|
| 概要 | ユーザーが支出金額とカテゴリを選択し記録する |
| 入力 | 金額（整数、カスタムキーパッド）、カテゴリ（タップ選択） |
| 出力 | トランザクション作成、予算残高の即時反映 |
| 対応画面 | SCR-002（入力モーダル）→ SCR-001（ダッシュボード） |
| 優先度 | Must（必須） |
| 受入条件 | ① 金額入力→カテゴリ選択→確定で予算残高が即座に減少する ② 金額0では記録できない ③ カスタムキーパッド上で四則演算が可能 |

### FR-002: 立替記録（IOU 2段入力）

| 項目 | 内容 |
|------|------|
| 概要 | 飲み会等で立替が発生した際、立替総額と自分の支出を分離して記録 |
| 入力 | 立替総額、自分の支出額（それぞれカスタムキーパッド）、カテゴリ |
| 出力 | 立替分（isIOU=true）と自己支出分（isIOU=false）の2トランザクション |
| 対応画面 | SCR-002（入力モーダル/立替モード） |
| 優先度 | Must（必須） |
| 受入条件 | ① 立替総額と自己支出を同一画面で入力できる ② 立替分は予算に影響しない ③ 自己支出分のみ予算から減算される ④ 自己支出未入力で確定するとバリデーションエラー ⑤ 立替総額 < 自己支出でバリデーションエラー |

### FR-003: 臨時収入記録

| 項目 | 内容 |
|------|------|
| 概要 | 給与・賞与・副業等の臨時収入を記録し、予算総額に加算する |
| 入力 | 金額、収入カテゴリ（給与/賞与/副業/投資/お小遣い/その他） |
| 出力 | 収入トランザクション作成、予算総額の増加 |
| 対応画面 | SCR-002（入力モーダル/収入モード） |
| 優先度 | Must（必須） |
| 受入条件 | ① 収入記録で予算総額（totalAmount）が増加する ② 収入カテゴリの選択ができる |

### FR-004: ダッシュボード表示

| 項目 | 内容 |
|------|------|
| 概要 | 今月の予算残高、カテゴリ別の消費状況、直近の支出履歴を一画面で表示 |
| 表示要素 | メイン円グラフ、カテゴリ別一覧、直近履歴リスト、日次推移グラフ |
| 対応画面 | SCR-001 |
| 優先度 | Must（必須） |
| 受入条件 | ① 予算残高が正確に表示される ② カテゴリごとの残高がリスト表示される ③ 直近の支出が時系列で表示される ④ 予算オーバー時に警告が表示される |

### FR-005: 支出履歴の管理

| 項目 | 内容 |
|------|------|
| 概要 | 過去の支出を一覧表示し、編集・削除が可能 |
| 操作 | タップで編集モーダル表示、左スワイプで削除 |
| 制約 | 固定費は履歴から編集・削除不可（設定画面で一元管理） |
| 対応画面 | SCR-003 → SCR-002（編集時） |
| 優先度 | Must（必須） |
| 受入条件 | ① 編集で新規追加されず既存が上書きされる ② 削除で予算が復元される ③ 固定費の行は削除操作が無効 ④ 立替(IOU)削除時は予算に影響しない |

### FR-006: カテゴリ管理

| 項目 | 内容 |
|------|------|
| 概要 | 支出カテゴリのCRUD（追加・編集・削除・並び替え） |
| 操作 | 設定画面から操作 |
| 対応画面 | SCR-008 |
| 優先度 | Must（必須） |
| 受入条件 | ① カテゴリの追加・名前変更・予算変更・削除が可能 ② ドラッグで並び替え可能 |

### FR-007: 予算・固定費・先取り貯金設定

| 項目 | 内容 |
|------|------|
| 概要 | 手取り額、固定費テンプレ、先取り貯金額を一元管理 |
| 連動 | 固定費はトランザクションとして動的同期 |
| 対応画面 | SCR-007 |
| 優先度 | Must（必須） |
| 受入条件 | ① 手取り額変更が予算計算に反映される ② 固定費追加/削除で予算とトランザクションが同期される |

### FR-008: 月次レビュー

| 項目 | 内容 |
|------|------|
| 概要 | 月末に支出を振り返り、次月予算をリセット |
| 処理 | 予算超過時は次月に借金として繰越 |
| 対応画面 | SCR-006 |
| 優先度 | Must（必須） |
| 受入条件 | ① 次月予算が新規作成される ② 前月オーバー分が次月の初期使用額に加算される |

### FR-009: 立替(IOU)管理

| 項目 | 内容 |
|------|------|
| 概要 | 未回収・回収済みの立替一覧、メモ追加、回収ボタン |
| 対応画面 | SCR-005 |
| 優先度 | Must（必須） |
| 受入条件 | ① 未回収の立替が一覧表示される ② 回収ボタンで回収済みに変更可能 ③ メモの追加・編集が可能 |

### FR-010: ウィジェット表示

| 項目 | 内容 |
|------|------|
| 概要 | ホーム画面ウィジェットで残高と色ゲージを常時表示 |

| 対応画面 | SCR-010 |
| 優先度 | Must（必須） |

### FR-012: プッシュ通知

| 項目 | 内容 |
|------|------|
| 概要 | ユーザーのアプリ利用継続を促進するローカル通知機能 |
| フレームワーク | UserNotifications (UNUserNotificationCenter) |
| 対応画面 | 設定画面（SCR-007）で通知ON/OFF切り替え |
| 優先度 | Should（次フェーズ） |

#### 通知タイミング・内容一覧

| ID | トリガー | 送信タイミング | 通知内容 | 備考 |
|----|---------|--------------|---------|------|
| NOTIF-001 | 毎日リマインダー | 毎日21:00（設定で変更可） | 「今日の支出を記入しましたか？忘れないうちに記録しましょう 📝」 | その日の記録が0件の場合のみ送信 |
| NOTIF-002 | 月次レポート完成 | 月初1日 9:00 | 「先月の振り返りレポートが完成しました！確認してみましょう 📊」 | 前月のBudgetデータがある場合のみ |
| NOTIF-003 | 予算警告（80%消費） | 支出記録時にリアルタイム判定 | 「今月の予算の80%を使いました。残り¥XX,XXXです ⚠️」 | 月中1回のみ |
| NOTIF-004 | 予算オーバー | 支出記録時にリアルタイム判定 | 「今月の予算をオーバーしました！支出を見直しましょう 🚨」 | 月中1回のみ |
| NOTIF-005 | 立替未回収リマインダー | 毎週月曜 10:00 | 「未回収の立替が¥XX,XXXあります。回収を忘れずに 💰」 | 未回収IOUがある場合のみ |
| NOTIF-006 | 3日間未利用 | 最終操作から72時間後 | 「最近の支出を記録しましたか？3日分まとめて入力しても大丈夫です 👋」 | エンゲージメント回復用 |

#### 通知許可の取得フロー

1. **オンボーディング時（TutorialView完了後）**: 通知許可ダイアログを表示
   - システムの `requestAuthorization(options: [.alert, .badge, .sound])` を実行
   - 拒否された場合はそのまま進行（強制しない）
2. **設定画面**: 通知のON/OFF切り替えトグル（アプリ内制御）
   - OFF時はスケジュール済みの通知を全てキャンセル
   - ON時は通知スケジュールを再登録
3. **システム設定誘導**: アプリ内でONにしているのにシステム側で拒否されている場合、設定アプリへの誘導バナーを表示

#### 実装方針

| 項目 | 方針 |
|------|------|
| 通知の種類 | 全てローカル通知（サーバー不要） |
| スケジューリング | `UNCalendarNotificationTrigger` (定期) / `UNTimeIntervalNotificationTrigger` (遅延) |
| 管理クラス | `NotificationService` をServices層に追加 |
| 永続化 | 通知設定は `UserDefaults` で管理（通知有効/無効、リマインダー時刻） |
| バッジ | 未確認の月次レポートがある場合にバッジ数1を表示 |
| 受入条件 | ① 許可ダイアログがオンボーディング完了後に1回表示される ② 設定画面で通知ON/OFFが切り替えられる ③ 条件を満たした場合のみ通知が配信される ④ OFF時に通知が来ない |

## 6. 用語定義

| 用語 | 定義 |
|------|------|
| IOU | I Owe You の略。立替金（他者への貸し）を指す |
| 立替総額 | 飲み会等で全員分を支払った合計金額 |
| 自己支出分 | 立替総額のうち自分が負担する金額 |
| 実立替金 | 立替総額 − 自己支出分（他者から回収すべき金額） |
| 固定費 | 毎月定額で発生する費用（家賃等）。テンプレートから自動生成 |
| 先取り貯金 | 予算計算時に手取りから先に差し引かれる貯金額 |
| 借金繰越 | 前月の予算超過分を次月の初期使用額として加算する仕組み |
| No-Look Experience | アプリを開かずにウィジェットで予算状況を確認する体験 |
