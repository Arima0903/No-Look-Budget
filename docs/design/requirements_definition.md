## 1. プロジェクト概要

### 1.1 背景・目的

浪費家やADHD傾向のユーザーが継続可能な家計管理を実現するiOSアプリ。
既存の家計簿アプリが持つ「入力の面倒さ」「立替による管理崩壊」を解消し、
**アプリを開かなくても予算状況が把握できる（No-Look Experience）** を提供する。

### 1.2 対象ユーザー

- 浪費癖のある20〜40代（主にiPhoneユーザー）
- 入力作業が継続できないADHD傾向のユーザー
- 飲み会やグループ活動で立替が頻発するユーザー

### 1.3 ビジネスゴール

- App Storeでの公開・リリース
- サブスクリプション/機能課金によるマネタイズ

## 2. 機能要件

### FR-001: 支出記録

| 項目 | 内容 |
|------|------|
| 概要 | ユーザーが支出金額とカテゴリを選択し記録する |
| 入力 | 金額（整数、カスタムキーパッド）、カテゴリ（タップ選択） |
| 出力 | トランザクション作成、予算残高の即時反映 |
| 優先度 | 必須 |

### FR-002: 立替記録（IOU 2段入力）

| 項目 | 内容 |
|------|------|
| 概要 | 飲み会等で立替が発生した際、立替総額と自分の支出を分離して記録 |
| 入力 | 立替総額、自分の支出額（それぞれカスタムキーパッド）、カテゴリ |
| 出力 | 立替分（isIOU=true）と自己支出分（isIOU=false）の2トランザクション |
| 優先度 | 必須 |

### FR-003: 臨時収入記録

| 項目 | 内容 |
|------|------|
| 概要 | 給与・賞与・副業等の臨時収入を記録し、予算総額に加算する |
| 入力 | 金額、収入カテゴリ（給与/賞与/副業/投資/お小遣い/その他） |
| 出力 | 収入トランザクション作成、予算総額の増加 |
| 優先度 | 必須 |

### FR-004: ダッシュボード表示

| 項目 | 内容 |
|------|------|
| 概要 | 今月の予算残高、カテゴリ別の消費状況、直近の支出履歴を一画面で表示 |
| 表示要素 | メイン円グラフ、カテゴリ別一覧、直近履歴リスト、日次推移グラフ |
| 優先度 | 必須 |

### FR-005: 支出履歴の管理

| 項目 | 内容 |
|------|------|
| 概要 | 過去の支出を一覧表示し、編集・削除が可能 |
| 操作 | タップで編集モーダル表示、左スワイプで削除 |
| 制約 | 固定費は履歴から編集・削除不可（設定画面で一元管理） |
| 優先度 | 必須 |

### FR-006: カテゴリ管理

| 項目 | 内容 |
|------|------|
| 概要 | 支出カテゴリのCRUD（追加・編集・削除・並び替え） |
| 操作 | 設定画面から操作 |
| 優先度 | 必須 |

### FR-007: 予算・固定費・先取り貯金設定

| 項目 | 内容 |
|------|------|
| 概要 | 手取り額、固定費テンプレ、先取り貯金額を一元管理 |
| 連動 | 固定費はトランザクションとして動的同期 |
| 優先度 | 必須 |

### FR-008: 月次レビュー

| 項目 | 内容 |
|------|------|
| 概要 | 月末に支出を振り返り、次月予算をリセット |
| 処理 | 予算超過時は次月に借金として繰越 |
| 優先度 | 必須 |

### FR-009: 立替(IOU)管理

| 項目 | 内容 |
|------|------|
| 概要 | 未回収・回収済みの立替一覧、メモ追加、回収ボタン |
| 優先度 | 必須 |

### FR-010: ウィジェット表示

| 項目 | 内容 |
|------|------|
| 概要 | ホーム画面ウィジェットで残高と色ゲージを常時表示 |
| 連携 | App Group経由でSwiftDataコンテナを共有 |
| 優先度 | 必須 |

### FR-011: プレミアム機能（将来）

| 項目 | 内容 |
|------|------|
| 概要 | 借金分割プラン等のプレミアム機能 |
| 制御 | `isPremiumEnabled` フラグ |
| 優先度 | 将来 |

## 3. 非機能要件

### NFR-001: セキュリティ

- APIキー・個人情報はKeychainで管理
- ソースコードへのハードコーディング禁止

### NFR-002: パフォーマンス

- 入力から予算反映まで1秒以内
- ウィジェットの表示更新はWidgetKit制約に準拠

### NFR-003: ユーザビリティ

- 金額入力は全てカスタムキーパッド（標準キーボード使用禁止）
- ダークモード対応
- 触覚フィードバック（Haptics）全操作に適用

### NFR-004: 対応OS

- iOS 17.0以降

### NFR-005: データ永続化

- SwiftData + App Group によるアプリ・ウィジェット間共有

## 4. 制約事項

| ID | 制約 |
|----|------|
| CON-001 | Apple Pay/Wallet連携はAPI制約によりスコープ外 |
| CON-002 | WidgetKitの更新頻度はOSのPolicy制限に従う |
| CON-003 | 金額は整数（円単位）で管理する |

## 5. 用語定義

| 用語 | 定義 |
|------|------|
| IOU | I Owe You の略。立替金（他者への貸し）を指す |
| 立替総額 | 飲み会等で全員分を支払った合計金額 |
| 自己支出分 | 立替総額のうち自分が負担する金額 |
| 実立替金 | 立替総額 − 自己支出分（他者から回収すべき金額） |
| 固定費 | 毎月定額で発生する費用（家賃等）。テンプレートから自動生成 |
