## 1. システム構成

### 1.1 アーキテクチャパターン

**MVVM + Repository(Service) Pattern** を採用。

```
┌──────────────────────────────────────────────────┐
│  View層                                          │
│  (SwiftUI Views / Widget)                        │
├──────────────────────────────────────────────────┤
│  ViewModel層                                     │
│  (ObservableObject / AppIntent)                  │
├──────────────────────────────────────────────────┤
│  Service層                                       │
│  (TransactionServiceProtocol)                    │
├──────────────────────────────────────────────────┤
│  Model層 (SwiftData @Model)                      │
│  Budget / ItemCategory / ExpenseTransaction /    │
│  IOURecord / FixedCostSetting                    │
├──────────────────────────────────────────────────┤
│  App Group (共有ストレージ)                       │
│  group.com.arima0903.NoLookBudget                │
└──────────────────────────────────────────────────┘
```

## 4. 入力バリデーション設計

### 4.1 通常支出入力（FR-001）

| バリデーションID | ルール | エラーメッセージ |
|-----------------|--------|----------------|
| VLD-001 | 金額 > 0 であること | （確定ボタン無効化） |
| VLD-002 | カテゴリが選択されていること | （初期選択あり） |
| VLD-003 | 計算式が有効であること（末尾演算子不可、連続演算子不可） | （計算結果非表示） |

### 4.2 立替入力（FR-002）

| バリデーションID | ルール | エラーメッセージ |
|-----------------|--------|----------------|
| VLD-101 | 自分の支出欄が入力済みであること（初期値 "0" のままは未入力扱い） | 「自分の支出額を入力してください」 |
| VLD-102 | 立替総額 ≧ 自分の支出額 であること | 「立替総額が自分の支出額より小さくなっています」 |
| VLD-103 | 立替総額 > 0 であること | （確定ボタン無効化） |

> [!IMPORTANT]
> VLD-101では「本当に自分の支出が0円」のケースに対応するため、`"0+0"` など式として入力された場合は入力済みとみなす。初期値 `"0"` と区別するための設計判断。

### 4.3 臨時収入入力（FR-003）

| バリデーションID | ルール | エラーメッセージ |
|-----------------|--------|----------------|
| VLD-201 | 金額 > 0 であること | （確定ボタン無効化） |
| VLD-202 | 収入カテゴリが選択されていること | （初期選択あり） |

## 5. 予算計算ロジック

### 5.1 支出記録時の予算更新

1. `Budget.spentAmount += amount`
2. `ItemCategory.spentAmount += amount`（カテゴリ指定時）
3. 立替（isIOU=true）の場合、`Budget.spentAmount` は増加しない

### 5.2 立替2段入力の計算ロジック

```
入力値:
  立替総額     = iouExpression の計算結果
  自分の支出額 = myExpenseExpression の計算結果

計算:
  実立替金 = 立替総額 − 自分の支出額

保存:
  if 実立替金 > 0:
    ExpenseTransaction(amount: 実立替金, isIOU: true)   → 予算に影響しない
  if 自分の支出額 > 0:
    ExpenseTransaction(amount: 自分の支出額, isIOU: false) → 予算から減算
```

### 5.3 月跨ぎ処理

1. 前月の `spentAmount > totalAmount` の場合、差額を借金として計算
2. 次月の `Budget.totalAmount` から借金額を差し引いた予算でスタート
