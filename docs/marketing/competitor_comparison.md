# 競合比較表（No-Look-Budget vs 既存サービス）

本プロジェクト（No-Look-Budget）が、既存の巨大な家計簿アプリ（マネーフォワード ME、Zaim）とどのように差別化されているかを整理した表です。
「高機能さ」で勝負するのではなく、「ターゲットユーザー（浪費家・面倒くさがり）のペインを解決することに特化した引き算のUI/UX」を強みとしています。

| 機能・体験・特徴 | No-Look-Budget (本作) | マネーフォワード ME | Zaim |
| :--- | :--- | :--- | :--- |
| **メイン・ターゲット** | 浪費家、ADHD傾向、ズボラで続かない人 | 几帳面な人、資産全体の把握・運用をしたい人 | 節約志向の人、日々の家計簿を細かくつけたい人 |
| **最重要UI（コア体験）** | **ロック画面/ホーム画面の巨大ウィジェット** | アプリ内の資産推移グラフ・口座一覧画面 | アプリ内のタイムライン・カテゴリ別支出画面 |
| **残高の「見せ方」** | アプリを**開かせず**、ウィジェットで「残り予算」を強制視界占有（残量で警告色へ変化） | アプリを開いて、連携された金融機関のデータから詳細なレポートを確認する | アプリを開いて、日々の支出記録や予算比をグラフで振り返る |
| **支出入力のハードル** | **極小**: ウィジェットタップで即座に数字入力（またはApple Wallet連携で全自動） | **小**: 銀行・クレカ等の自動連携がメイン（手入力するには数タップ必要） | **小〜中**: 手入力、レシートOCR、銀行・クレカ連携（手入力UIは比較的使いやすい） |
| **自動連携のメンタル負荷** | Apple Wallet (Apple Pay) 決済のみに絞るため、**連携切れによる再認証のストレスがない** | 多種多様な連携が可能だが、定期的にパスワード変更等で連携が切れ、修復がストレスになる | マネーフォワードと同様、API連携の有効期限などで設定し直す手間が発生しうる |
| **立替金（ノイズ）の処理** | **立替セパレーター**: スワイプ一発で別枠へ退避。自分のメイン予算（完璧な状態）を綺麗に保つ | 「振替」や「計算対象外」設定はあるが、操作が煩雑で予算グラフにノイズが乗りやすい | 立替機能はあるが、手動での調整・相殺入力が必要で手間がかかり放置されがち |
| **カテゴリ（費目）自動分類** | **大雑把・最小限**（食費、交際費、交通費、その他 程度）。目的は「今月の残高把握」のみ | 高精度。AIが「外食」「カフェ」「日用品」など細かく自動仕分けしてくれる | 高精度。ユーザー独自のカテゴリカスタマイズも豊富 |
| **レシートOCR（撮影）** | **なし（あえて削る）**。ターゲットはどうせ撮影する習慣が根付かないため。 | あり（読み取り精度が高い） | あり（読み取り精度が高い、独自機能も多い） |
| **マネタイズの主軸・動機** | 「強力な警告テーマ（激しいUI）」のアンロックや、広告非表示による**快適な自制心の購入** | 「連携口座数の無制限化」「1年以上の過去データ閲覧」など、**資産管理の高度化** | 「複数アカウント管理」「広告非表示」など、**家族入力や利便性の向上** |

## まとめ：勝てる理由（勝ち筋）

マネーフォワードやZaimは「全てのお金の流れを正確に記録・分析するツール」としては完璧ですが、それゆえに**「情報量が多すぎて見たくなくなる」「連携切れや立替などのイレギュラーで家計簿が崩れ、完璧主義が破綻して挫折する」**という弱点があります。

一方 No-Look-Budget は、**「今月あといくら使えるか（生存確認）」**だけを、アプリを開くことなく（= No-Look）強制的にユーザーの視界に叩き込みます。面倒なカテゴリ分けやレシート撮影も不要であり、立替ノイズもワンアクションで消せるため、**「過去にあらゆる家計簿アプリに挫折した浪費家」**にとって唯一使い続けられる（というか強制的に見せつけられる）アプリという圧倒的な差別化が図れています。
