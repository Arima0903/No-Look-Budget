# No-Look-Budget UI/UX 設計書（Draft）

## 1. 画面構成の基本方針
* **Apple純正アプリのような上質さ**: ダークモードを基調とし、鮮やかなネオンカラー（緑・赤・オレンジ）でステータスを直感的に伝える。
* **情報量の徹底排除**: 「残量ゲージ」と「金額」以外の不要なテキストや装飾は極力排除する。

## 2. コア画面と遷移フロー

### ① iOS ウィジェット（Lock/Home Screen）
* **見た目**: アプリアイコンサイズ（Small）〜横長（Medium）サイズ。
* **表示内容**:
  * 円形の残量ゲージ（提供画像の中心部のようなデザイン）。
  * 大きく「残り予算（Remaining）」の数字。
* **ギミック**: 予算が減るにつれて、ゲージと数字の色が「緑 → 黄色 → 警戒のオレンジ → 激しい赤」へと変化する。
* **遷移**: ウィジェット全体をタップすると、アプリ本体の「1タップ入力画面（ホーム）」が起動する。

### ② アプリ ホーム画面（1タップ入力 ＆ アプリ内ウィジェット）
* **見た目**: 提供していただいた画像のデザインをベースにする。
  * 画面上部〜中央: **「全体の予算」**を示す巨大な円形ゲージと残額（緑/赤）。
  * 画面下部: **「予算項目（カテゴリ）ごとの小ウィジェット（丸形）」**が横並び、あるいはグリッドで並ぶ。（画像下部の POKER, NOMIKAI, LENS など）
* **操作フロー（全体からの入力）**:
  1. ウィジェットから起動後、画面中央の「QUICK-SYNC & LOG（手動入力ボタン）」をタップ。
  2. 下からハーフモーダル（または全画面テンキー）がスッと立ち上がる。
  3. 金額を打ち込み、「使った」ボタンを1タップで記録完了（デフォルト設定の項目から引かれる）。
* **操作フロー（項目別ウィジェットからの入力・詳細確認）**:
  * 下部に並ぶ項目別ウィジェット（例：POKER）を**タップ**すると、その項目の詳細画面（履歴や予算設定）へ遷移する。
  * この項目別ウィジェットを**長押し（またはダブルタップ・特定アイコンのタップ）**すると、その項目（POKER）専用の入力モーダルが直接立ち上がる。

### ③ 立替セパレーター（スワイプ分離）の操作画面
* **課題**: 飲み会等の数万円の支払いを、メインの予算グラフの「赤字」として残さず、直感的に別枠へ逃したい。
* **操作案A: 「入力直後のスワイプアウェイ」**
  1. アプリを開き、通常通り「30000円」と入力する。
  2. 画面上に一時的に「30,000円使いました」というカード（履歴要素）が表示される。
  3. そのカードを**右に大きくスワイプ**するだけで、アニメーションと共に「立替（Front/IOU）プール」という別箱へ吸い込まれ、メインの残高が一瞬で元に戻る。
* **操作案B: 「入力時のトグルスイッチ」**
  1. 金額入力用のテンキー画面の下に、「💳 自分が払った（通常） / 🍻 立替（Front）」の巨大な切り替えスイッチ（またはタブ）を配置。
  2. 立替として入力すると、メインの予算グラフには一切触れず、下部の項目別ウィジェットの一つである「NOMIKAI (IOU)」などの立替専用ゲージに借金として追加される。
