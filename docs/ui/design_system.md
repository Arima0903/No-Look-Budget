# No-Look-Budget デザインシステム

## コンセプト
**「ADHD向け：開かなくてもわかる管理（No-Look Experience）」**
アプリでもウィジェットでも、数字を細かく読まなくても「感覚的」かつ「直感的」に現在の家計状況が把握できるUI/UXを提供する。

## カラーパレット

### 1. ゲージ・グラフの色（視覚的フィードバック）
グラフのゲージ色は以下の **2色のみ** とする。（オレンジなどは使用しない）
* **予算内（全体の枠、または残高）**: `Color(red: 0.4, green: 0.9, blue: 0.6)` （明るい緑 / 安全）
* **使用済み（消費分、立替含む）**: `Color(red: 0.9, green: 0.4, blue: 0.4)` （赤 / 危険・消費）
* **グラフ背景**: `Color(red: 0.4, green: 0.9, blue: 0.6).opacity(0.3)` （ベースの緑・半透明）

**【ルール】**
* すべての円グラフは「予算全体の緑色（ベース）」に対して、0時の方向から時計回りに「使用済みの赤色」が上書きされていく仕様とする。

### 2. テキスト・数字の色（状態フィードバック）
テキスト（特に残金金額を示す数字）の色は、現在の予算使用率（％）に応じて以下の **3段階** で変化させる。
* **0% 〜 49% (半分未満の使用)**: 緑色 `Color(red: 0.4, green: 0.9, blue: 0.6)`
* **50% 〜 99% (半分以上の使用)**: 黄色 `Color.yellow`
* **100%以上 (予算超過・マイナス)**: 赤色 `Color(red: 0.9, green: 0.4, blue: 0.4)`  ※マイナス表示は `-¥5000` のようにフォーマットする。

## レイアウトの統一ルール

### 1. ウィジェット表示 (WidgetKit)
* **情報の絞り込み**: ユーザーがウィジェットで一番知りたいのは「あといくら使えるか（残金）」であるため、ウィジェットのメイン円グラフ内には `SPENT（使用額）` は表示せず、`REMAINING（残金）` とその金額のみを大きく表示する。
* **余白の無効化**: iOS 17以降の `.contentMarginsDisabled()` を活用し、ウィジェット全体を黒背景で埋め尽くす。
* **文字被りの回避**: 円グラフの中に文字を配置する際は、文字が長くなった場合（桁数が多い場合）でもグラフの縁（ゲージ）に被らないように、`minimumScaleFactor` などを利用して動的にフォントサイズを縮小する設定を必ず導入する。

### 2. ダッシュボード表示 (App)
* **詳細な情報**: アプリを開いた際は、ウィジェットでは省略された詳細情報（SPENTや現在の具体的な進行度など）も合わせて表示する。
* **画面遷移**: 各円グラフはタップ可能にし、全体の履歴やカテゴリ別の履歴への `NavigationLink` として機能させる。
