## テストレベル別の役割

| テストレベル | 略称 | 目的 | 実施方法 |
|-------------|------|------|---------|
| Program Test | PT | 関数・クラス単位の動作検証 | XCTest (自動) |
| Integration Test | IT | 複数コンポーネント間のデータ連携検証 | XCTest + インメモリDB (自動) |
| System Test | ST | ユーザー視点のE2E操作検証 | シミュレータ/実機 (手動) |

---

## 観点カバレッジサマリ

| テストレベル | 観点数 | カバー率 |
|-------------|--------|---------|
| PT | 9/22 | 41% |
| IT | 11/22 | 50% |
| ST | 10/22 | 45% |
| **全レベル合計** | **22/22** | **100%** |

> [!NOTE]
> 各テスト観点は最低1つのテストレベルでカバーされています。
> PT で網羅的にロジックを検証し、IT でデータフロー整合性を確認し、ST でユーザー体験を保証する3層構造です。
