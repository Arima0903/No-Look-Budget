## 3. テストレベル別の役割

| テストレベル | 略称 | 目的 | 実施方法 | 自動化 |
|-------------|------|------|---------|:------:|
| Program Test | PT | 関数・クラス単位のロジック検証 | XCTest + インメモリDB | ✅ |
| Integration Test | IT | 複数コンポーネント間のデータ連携検証 | XCTest + インメモリDB | ✅ |
| System Test | ST | ユーザー視点のE2E操作検証 | シミュレータ/実機 | ❌ (手動) |

---

## 4. リスク重要度の定義

| リスク | 定義 | テスト基準 |
|:------:|------|----------|
| 高 | データ不整合や金額誤表示に直結する | PT + IT + ST の少なくとも2レベルで検証 |
| 中 | 機能の正確性に影響するが回避策がある | PT または IT の少なくとも1レベルで検証 |
| 低 | UXの快適性に影響するが機能不全ではない | ST での確認のみで可 |
