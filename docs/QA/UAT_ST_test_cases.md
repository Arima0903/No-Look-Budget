# No-Look-Budget ユーザー受け入れテスト (UAT) 仕様書

このドキュメントは、開発した「No-Look-Budget」がビジネス要件およびユーザーの期待する挙動を満たしているかを確認するためのテスト設計書です。
手元（シミュレータ・実機）でアプリを操作しながら、以下のテストケースを上から順番に実施してください。

---

## 🛡️ テスト環境・前提条件
- **テスト方法**: Xcodeのシミュレーター（iPhone 16 Pro等）または実機にアプリをビルドして実行。
- **データのリセット**: まっさらな状態からテストできるよう、可能であればアプリを一度削除（またはシミュレータ初期化）して再インストールしてから始めるのが理想的です。
- 初期データとして全体予算250,000円とカテゴリ別の予算がいくつか設定されている状態からスタートします。

---

## 🗂️ テストケース一覧

### 1. 【支出の追加・予算の連動確認】（基本フロー）

一番重要な「お金を使ったら全体予算とカテゴリ予算が正しく減るか」のテストです。

- **[ ] ケース1-1**:
    1. ホームのダッシュボードで、現在の「今月の残り予算（一番上の数字）」と「食費」カテゴリの残高をメモする。
    2. 「記録をつける」ボタンから入力モーダルを開く。
    3. カテゴリ「食費」を選択し、金額を `500` と入力し「使う」をタップ。
    4. **[期待結果]**:
       - モーダルが閉じる
       - ホームの「今月の残り予算」が 500円 減っていること
       - リストにある「食費」の残り予算が 500円 減っていること

### 2. 【立替機能の確認】（スワイプ分離）

飲み会などの立替分が、自分の予算を圧迫しないかをテストします。

※ 現状の仕様は後日変更予定ですが、現バージョンでのテストとして残します。

- **[ ] ケース2-1**:
    1. 入力モーダルを開く。
    2. 上部の「立替」トグルをONにする（デザインがオレンジに変わる）。
    3. 金額 `5000` を入力し「立替」をタップ。
    4. **[期待結果]**:
       - ホームの「今月の残り予算」は**減らない**こと
       - カテゴリを選択していた場合、そのカテゴリの残高も**減らない**こと
- **[ ] ケース2-1**:
    1. 入力モーダルを開く。
    2. 上部の「立替」トグルをONにする（デザインがオレンジに変わる）。
    3. 金額 `5000` を入力し、下の「自分の支出」欄に `1000` を入力し「立替」をタップ。
    4. **[期待結果]**:
        - ホームの「今月の残り予算」は**減らない**こと
        - カテゴリを選択していた場合、そのカテゴリの残高も**減らない**こと

- **[ ] ケース2-2**:
    1. 入力モーダルを開く。
    2. 立替トグルON。
    3. 金額 `3000` を入力し、自分の支出欄を空欄のまま「立替」タップ。
    4. **[期待結果]**:
        - アラート「自分の支出額を入力してください」が表示され、確定できない。
### 3. 【履歴の編集・更新確認】

入力した金額やカテゴリが間違っていた場合に、新しく追加されず「上書き」されるかのテストです。

- **[ ] ケース3-1**:
    1. ホーム画面の大きな円グラフをタップして「今月の全体予算」詳細を開く、またはメニューから「支出履歴を開く」。
    2. 先ほど入力した食費 `500` 円の履歴行をタップする。
    3. 編集モーダルが開かれたら、金額を `1000` に修正して「使う」をタップする。
    4. **[期待結果]**:
       - 新しい履歴が追加されないこと
       - 該当の履歴行の金額が `1000` 円に変わっていること
       - ホームの全体予算・食費予算から追加で 500円分（すでに500引かれているので合計1000分）のみ引かれた状態になること

- **[ ] ケース3-2**:
    1. 支出履歴から、どれか一つの支出（例: 食費 1000円）を**左にスワイプ**し、「削除（Delete）」をタップ。
    2. **[期待結果]**:
       - 履歴一覧から該当の行が消えること
       - ホームの「全体予算」と「カテゴリ予算（食費）」に 1,000円 が**戻っている**（増えている）こと

### 4. 【臨時収入の確認】

給与やプレゼントなどの収入を記録するテストです。

※ 収入は**全体予算には追加されません**。別途「予算・貯金の設定」から変更する思想です。

- **[ ] ケース4-1**:
    1. ダッシュボードの全体予算をメモする。
    2. 入力モーダルを開き、上部のセグメントで「臨時収入」を選択する（デザインが緑に変わる）。
    3. 「給与」を選び、金額に `50000` を入力して「追加」をタップ。
    4. **[期待結果]**:
       - 全体の「今月の残り予算」は **増えない** こと。
       - （あくまで履歴としてのみ残る）

### 5. 【月次レビューと借金繰越の確認】

月を跨いだときのリセット処理のテストです。

- **[ ] ケース5-1**:
    1. サイドメニューから「月末振り返り」をタップする。
    2. 現在の実際の支出額と予算が正しいか確認する。
    3. 「報酬を受け取って次月へ」ボタンをタップする。
    4. **[期待結果]**:
       - 月がリセットされ、ダッシュボードの表示が次の月（基本予算250,000円など）に戻ること
       - もし前月オーバーしていた場合、あらかじめ設定予算からそのオーバー分（借金）がマイナスされた状態でスタートしていること

### 6. 【予算オーバー警告の確認】


- **[ ] ケース6-1**:
    1. 入力モーダルから、今月の全体予算を超える支出を記録（例: 200万円を食費として記録）。
    2. ホーム画面上部に「予算をオーバーしています！」というテキストが表示されていることを確認する。
    3. **[期待結果]**:
       - （確認ボタンなどはなく）予算をオーバーしている旨だけが表示されること。

- **[ ] ケース6-2**:
    1. 前月の予算を超えているが、当月の予算は設定内の場合、「前月の借金により予算修正が必要です」というテキストが表示されることを確認。
    2. （※当月の予算がオーバーしているときには、来月の借金に関する警告は表示されない）
