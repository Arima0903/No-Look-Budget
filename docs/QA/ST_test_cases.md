## 1. テスト方針

STでは、ユーザー視点でのエンドツーエンド（E2E）操作を検証する。
シミュレータまたは実機でアプリを起動し、画面操作→データ結果を目視確認する。

---

## 2. テスト環境・前提条件

- **テスト方法**: Xcodeのシミュレーター（iPhone 16 Pro等）または実機
- **データのリセット**: アプリ削除→再インストールして初期状態からテスト
- 初期データとして全体予算250,000円とカテゴリ別の予算がいくつか設定されている状態

---

## 3. テストケース

### 3.1 支出の追加・予算の連動確認（基本フロー）

- **[ ] ケースST-001**: (テスト観点: TP-10, TP-19)
    1. ホームのダッシュボードで、現在の「今月の残り予算」と「食費」カテゴリの残高をメモ
    2. 「記録をつける」ボタンから入力モーダルを開く
    3. カテゴリ「食費」を選択し、金額を `500` と入力し「使う」をタップ
    4. **[期待結果]**:
        - モーダルが閉じる
        - ホームの「今月の残り予算」が 500円 減っている
        - リストにある「食費」の残り予算が 500円 減っている

### 3.2 立替機能の確認（2段入力）

- **[ ] ケースST-002**: (テスト観点: TP-12, TP-20)
    1. 入力モーダルを開く
    2. 上部の「立替」トグルをONにする（デザインがオレンジに変わる）
    3. 上段の「みんなの立替分」に金額 `5000` を入力し、下段の「自分自身の支出」をタップし `1000` を入力
    4. 「立替」をタップ
    5. **[期待結果]**:
        - ホームの「今月の残り予算」が1000円のみ減っている（5000円は減らない）
        - 直近の履歴に自分の支出（1000円）が表示されている

- **[ ] ケースST-003**: (テスト観点: TP-03, TP-21)
    1. 入力モーダルを開き、立替トグルON
    2. 金額 `3000` を入力し、自分の支出欄を空欄のまま「立替」タップ
    3. **[期待結果]**:
        - アラート「自分の支出額を入力してください」が表示され、確定できない

- **[ ] ケースST-004**: (テスト観点: TP-03, TP-21)
    1. 入力モーダルを開き、立替トグルON
    2. 上段に `1000`、下段に `3000` を入力して「立替」タップ
    3. **[期待結果]**:
        - アラート「立替総額が自分の支出額より小さくなっています」が表示され、確定できない

### 3.3 履歴の編集・更新確認

- **[ ] ケースST-005**: (テスト観点: TP-13)
    1. 支出履歴画面を開く
    2. 先ほど入力した食費 `500` 円の履歴行をタップ
    3. 編集モーダルが開かれたら、金額を `1000` に修正して「更新」をタップ
    4. **[期待結果]**:
        - 新しい履歴が追加されない
        - 該当の履歴行の金額が `1000` 円に変わっている
        - ホームの全体予算・食費予算から合計1000分が引かれた状態

- **[ ] ケースST-006**: (テスト観点: TP-14)
    1. 支出履歴から、どれかの支出を左にスワイプし、「削除」をタップ
    2. **[期待結果]**:
        - 履歴から該当の行が消える
        - ホームの全体予算とカテゴリ予算にその金額が戻っている

### 3.4 臨時収入の確認

- **[ ] ケースST-007**: (テスト観点: TP-09)
    1. ダッシュボードの全体予算をメモ
    2. 入力モーダルを開き、「臨時収入」を選択
    3. 「給与」を選び、金額に `50000` を入力して「追加」をタップ
    4. **[期待結果]**:
        - 全体の「今月の残り予算」が 50000円 増えること

### 3.5 月次レビューと借金繰越の確認

- **[ ] ケースST-008**: (テスト観点: TP-16)
    1. サイドメニューから「月末振り返り」をタップ
    2. 「報酬を受け取って次月へ」ボタンをタップ
    3. **[期待結果]**:
        - 月がリセットされ、ダッシュボードが初期予算に戻る
        - 前月オーバーの場合、借金分がマイナスされてスタート

### 3.6 予算オーバー警告の確認

- **[ ] ケースST-009**: (テスト観点: TP-22)
    1. 全体予算を超える支出を記録
    2. **[期待結果]**:
        - ホーム画面に「予算をオーバーしています！」のテキスト表示
