# Orbit Budget 知的財産・著作権・商標管理

本アプリで扱うデザインリソース、フォント、および他社の権利物（IP）の管理ポリシーです。
App Store審査の厳密な Guideline 5.2（知的財産権）を遵守し、トラブルを未然に防ぎます。

## 1. フォントライセンス
本アプリの「巨大な数字表示」等はUIのコアとなるため、フォント選びが重要になります。

* **原則**: Apple純正のシステムフォント（San Francisco 等）をそのまま利用するか、商用利用可能なフリーフォント（Google Fonts 等）のみを使用します。
* **システムフォントの扱い**: iOSのシステムフォントはアプリ内に埋め込む必要がなく、ライセンス上も最も安全です。
* **Google Fonts等の利用時**: 例え無料であっても、SIL Open Font License (OFL) などのライセンス明記が求められるため、上述の `oss_licenses.md` で管理し、アプリ内にクレジット表記を実装します。

## 2. 画像・アイコンの著作権と肖像権
* **システムアイコン**: Apple純正の [SF Symbols](https://developer.apple.com/sf-symbols/) を積極的に活用します。これはiOSへの組み込みにおいて最も安全かつ美しいデザインを提供します。
* **外部アイコン・画像の利用**:
  * フリー素材を使用する場合でも、必ず「商用利用可否」「クレジット表記の要否」を確認します。
  * いわゆる「AI生成画像」を使用する場合、生成元AIの規約で商用利用が可能か確認し、特定の既存キャラクター等に似ていないか（著作権侵害の恐れがないか）をレビューします。
  * アプリ内で個人の写真（肖像権）を要求・使用する機能は現段階では実装しません。

## 3. 商標権（App Store メタデータ）
* **アプリ名とサブタイトル**: 「Orbit Budget」という名称で進めますが、ストア公開前に類似商標（特に家計簿ジャンル）がないか、特許庁のJ-PlatPat等で簡易調査を推奨します。
* **競合の排除**: App Storeのキーワードや説明文に、話題性目的で「マネーフォワードより簡単」「Zaimの代わりに」といった**他社商標（サービス名）を記載することは固く禁じられています。**（即リジェクト対象です）

## 4. 競合アプリからの模倣（コピーキャット）防衛策 【重要】
「開かせないUX」「立替スワイプ分離」といった本アプリのコアバリューは、技術的難易度よりも「アイデアの秀逸さ」にあるため、資金力のある競合アプリ（Zaimやマネーフォワード等）に機能として模倣されるリスクが高いです。これを法務・知財的にどう防ぐ（あるいは牽制する）かが重要になります。

### 🛡 具体的な防衛・牽制策
1. **意匠権（UIデザインの特許）の検討**
   * 特に「巨大な円形ゲージからスワイプで立替プールへ飛ばす」といった特徴的なUI/操作フローについては、アプリリリース前に特許庁へ「画像意匠（画面デザインの権利）」の出願を検討します。
   * 意匠権を取る（または「出願中」と明記する）ことで、大企業がそのままUIをパクることを強力に牽制できます。
2. **ビジネスモデル特許の検討**
   * 「Apple Walletの決済通知をトリガーにし、立替フラグを自動判定して予算残高をUIに反映するシステム」など、複数の技術を組み合わせた独自の処理フローがある場合、特許の対象になり得ます。
3. **ブランド・コンセプトの先行認知（マーケティング防衛）**
   * 「Orbit Budget（仮）」というネーミングと、「ADHD/浪費家向けの立替分離家計簿」という独自のコンセプト（ポジショニング）を早期に商標登録し、SNS等で「私たちが元祖である」という認知を広めることが最大の防御になります。
4. **コードの難読化（Obsfucation）**
   * 審査には関係ありませんが、独自の「残高計算アルゴリズム」や「色を劇的に変える閾値ロジック」などは、リバースエンジニアリングで簡単に盗まれないよう、リリース時に最低限の難読化処置を施します。
