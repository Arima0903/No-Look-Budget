# 技術検証（PoC）レポート

## 1. Apple Wallet (Apple Pay) 決済連携のフック可否
**結論: 完全な「バックグラウンドでのリアルタイム即時反映」は技術的制約・審査のハードルが高く、段階的なアプローチが必要。**

### 技術的詳細と現状
* AppleはiOS 17.4より **`FinanceKit`** を拡張し、ユーザーの同意のもとクレジットカードや銀行のトランザクションデータにアクセスするAPIの提供を開始しました（主に欧州・米国から順次拡大）。
* しかし、**「ユーザーがコンビ二でApple Pay決済をした瞬間に、サードパーティアプリ（本アプリ）がバッググラウンドで即座にその決済イベントと金額をフックし、ローカルDBの数値を書き換えてウィジェットを更新する」**という完全なPush型のリアルタイム処理は、セキュリティおよびプライバシーの観点から現状のパブリックAPIでは非公開（またはエンタイトルメントの取得が極めて困難）です。

### 代替案（ピボット方針）
Apple Walletの自動連携はMVPリリース（初回）ではスコープから一旦外し、以下の「ローフリクション入力」でカバーします。
1. **Interactive Widget (iOS 17+) による超速入力**: アプリを開かなくても、ウィジェット上のボタンをタップするだけで入力を完了させる。
2. **iOS Shortcuts（ショートカットアプリ）の活用**: 「Apple Payで支払いをした時」というオートメーションのトリガーをユーザー自身で設定してもらい、本アプリのAppIntent（引数:金額）を裏で呼び出すハック的な自動化。

## 2. WidgetKit (iOS 17/18) によるダイナミックUI更新
**結論: 実現可能。Interactive Widgetを用いることで「No-Look」体験のコアを構築できる。**

### 技術的詳細と現状
* iOS 17から導入された **`Interactive Widgets` (AppIntent)** を利用することで、ウィジェット上のボタン（例: 「-1000円」ボタンや「入力モードへ」ボタン）をタップした際に、アプリ本体を起動することなくバックグラウンドで処理（SwiftDataのデータ更新）を走らせることができます。
* 処理が完了すると、WidgetKitのタイムラインが即座にリロードされ、残高やグラフの色がダイナミックに変化します。

### 実装方針
* ウィジェットへの配置: 残高を大きく表示し、その下に「よく使う金額（例: 500, 1000）」のワンタップボタン、またはアプリの入力モーダルへ直接飛ぶ（DeepLink）ボタンを配置します。
* 残高が特定の閾値（例えば残り20%）を切った場合、ウィジェットの背景色全体を「赤」等の警告色に切り替える処理をTimelineProviderに実装します。
